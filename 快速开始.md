# 自动 Sitemap 生成功能 - 快速开始

## 🎉 现在发布文章更简单了！

不再需要手动配置路由，只需创建 Markdown 文件即可！

## 📝 发布新文章的步骤

### 1. 创建文章文件

在适当的目录下创建 `.md` 文件：

```bash
docs/guides/chatgpt/my-new-article.md
```

### 2. 添加文章内容

**方式一：使用 frontmatter（推荐）**

```markdown
---
title: ChatGPT 最新使用指南
changefreq: daily
priority: 1.0
---

# 文章正文内容...
```

**方式二：只写标题**

```markdown
# ChatGPT 最新使用指南

文章正文内容...
```

### 3. 构建并生成 sitemap

```bash
pnpm docs:build
```

就这么简单！系统会自动：

- ✅ 扫描你的新文章
- ✅ 添加到 sitemap.xml
- ✅ 添加到侧边栏
- ✅ 根据路径自动分类

## 🔧 Frontmatter 配置说明

### title（标题）

文章标题，如果不设置：
1. 会从文档第一个 `#` 标题提取
2. 如果还没有，使用文件名

### changefreq（更新频率）

```yaml
changefreq: daily    # 每天更新（推荐用于博客和指南）
changefreq: weekly   # 每周更新（默认值）
changefreq: monthly  # 每月更新
```

### priority（优先级）

```yaml
priority: 1.0   # 最高优先级（首页）
priority: 0.9   # 重要文章（ChatGPT 主要指南）
priority: 0.8   # 一般文章（默认值）
priority: 0.7   # 次要文章
```

**💡 提示：** 如果不设置，系统会根据路径自动判断合适的优先级！

## 🚫 哪些文件会被排除？

以下文件/目录不会出现在 sitemap 中：

- `.vitepress/` 目录
- `node_modules/` 目录
- `script/` 目录
- `public/` 目录
- `README.md` 文件
- `CLAUDE.md` 文件

## 📊 构建结果

运行 `pnpm docs:build` 后，会在 `docs/.vitepress/dist/` 生成：

```
dist/
├── sitemap.xml      # XML 格式（搜索引擎用）
├── sitemap.txt      # TXT 格式（URL 列表）
├── sitemap.html     # HTML 格式（用户浏览）
└── robots.txt       # 搜索引擎爬虫配置
```

## 💡 常见问题

### Q: 我的文章没有出现在 sitemap 中？

A: 检查：
1. 文件扩展名是否为 `.md`
2. 文件是否在排除目录中
3. 重新运行 `pnpm docs:build`

### Q: 标题显示为文件名怎么办？

A: 添加 frontmatter 中的 `title` 或在文档开头添加 `#` 标题

### Q: 如何修改自动分类规则？

A: 编辑 `docs/.vitepress/utils/routes.ts` 中的 `getRoutesByCategory()` 函数
